Making a website

Project 1

Project Description

For this project we should create a website with the purpose of documenting the projects in this course. The website should also include information about yourself and your resume. It should be created by using HTML and then uploaded to GitHub using Git Bash.

Preface

Prior to the making of this website I had little previous experience with web page development and no experiance with HTML. I wanted to make a website I could use sometime in the future as a portfolio for my work and projects. Therefor, I created this webpage for that purpose and aswell to document my work for the course Manufacturing Processes I am attending this semester.

Creating the website and its repository

1. Preperation

First, because of my lack of knowledge of the matter of creating a website using HTML, I began by quickly scrolling through the HTML tutorial on w3schools.com/html. Again, because of lack of knowledge and because of convenience, I decided to use a templete for my website. As recommended by Hafliði I searched for a suitable templete at html5up.net. I choose the templete "Landed" because of its simple and clean style, which I felt was well suited for my purpose.



2. Website creation

To create the website an enviroment for web development is needed. I used the text editor Brackets. In Brackets I changed the HTML files for my templete of choice to suit the website for my needs. To do that I mainly used the HTML files that came with the templete and created new ones where I added or deleated sections from the old ones. For additional information I relied on w3schools.com/html and Google.

3. Creating a repository and uploading to GitHub

When I was happy with the overall look of the website the next step was to create a repository through GitHub and upload it to the repository with Git Bash. Git Bash can be downloaded here.

First create a repository on GitHub:



Next Git Bash is used to upload all of the files needed for the website to GitHub. I had trouble using the insturcions on GitHub, only "README.MD" was uploaded to my repository. Therefor, I used the following commands in Git Bash:

  1. Open Git Bash and enter your user name and email:
    git config --global user.name "your_username"
    git config --global user.email "your_email_address@example.com"
  2. Go to your website's directory by using the cd command in Git Bash or right click on "Git Bash Here" in your website's folder.
  3. Write the following commands:
    git init
    git add .
    git commit -m "first commit"
    git branch -M main
    git remote add origin https://github.com/Valgeir99/valgeir.git
    git push -u origin main


The website is now in your repository.

Finally, to publish your website go to Settings->GitHub Pages->Source->Branch: main



If you want to update your repository the following commands in Git Bash are used when you are in your website's directory on your computer:
git add .
git commit -m "what did you change?"
git push"
